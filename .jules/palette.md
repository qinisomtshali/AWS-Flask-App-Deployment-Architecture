## 2026-02-22 - Accessible and Optimized Visual Documentation
**Learning:** In documentation-heavy repositories, the "UI" is the documentation itself. Visual aids like architecture diagrams significantly improve the intuitive understanding of the system, but they must be accessible (alt text) and optimized (explicit dimensions to prevent layout shifts) to provide a first-class user experience.
**Action:** Always ensure that images in documentation have descriptive alt text and explicit width/height attributes to prevent CLS (Cumulative Layout Shift).
