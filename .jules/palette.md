## 2025-03-14 - Semantic Documentation as UI

**Learning:** In technical repositories, the README is often the primary user interface. Using semantic HTML tags like `<figure>` and `<figcaption>` for architecture diagrams provides critical context for screen readers and improves the visual hierarchy for all users. Properly naming assets (e.g., `architecture-diagram.webp` instead of generic or generated filenames) also improves long-term maintainability.

**Action:** Always prefer semantic `<figure>`/`<img>`/`<figcaption>` blocks over standard Markdown image syntax for complex technical diagrams to ensure accessibility and professional presentation.
