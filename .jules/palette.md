# Palette's Journal

## 2025-02-25 - Semantic Diagrams for Documentation
**Learning:** In documentation-heavy repositories, technical diagrams are the primary user interface. Using semantic HTML like `<figure>` and `<figcaption>` instead of just `<img>` provides better context for screen readers and improves the overall accessibility of the documentation.
**Action:** Always wrap key architecture diagrams in `<figure>` and use `<figcaption>` to provide a clear, accessible description that complements the `alt` text.
